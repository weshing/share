# 新标签页设置
## 1. 将书签设置应用到全部

通过编辑单个书签，可以实现共享一个书签的外观和感受。书签编辑模式有三个选项可以将设置复制到其他书签。这些复选框可在“视觉与名称”、“布局”和“主题”标签页底部找到。

当勾选这三个选项中的任何一个时，被编辑的书签设置会被复制到所有其他书签上。

### 将“显示可视化元素”和“显示名称”应用到其他书签上


**将被复制的设置：**

-   显示或隐藏视觉元素。不是实际输入的信件、图标或图片 URL
-   显示或隐藏名字。不是输入的实际姓名文字

**不会被复制的设置：**

-   信件作为视觉元素进入
-   被选为视觉元素的图标
-   作为视觉元素输入的图片 URL 是

### 将布局应用到其他书签



**将被复制的设置：**

-   视觉元素的大小和名称
-   可视化元素对齐到书签角落或中心
-   视觉元素和名称的水平偏移、垂直偏移和旋转
-   可视化元素和名称的堆叠顺序和方向
-   视觉元素与名称之间的槽空间

**不会被复制的设置：**

-   书签 宽瓦片
-   收藏高塔格

### 将主题应用到其他书签



**将被复制的设置：**

-   定制配色
-   定制口音
-   书签颜色的不透明度
-   书签边境
-   视觉元素阴影

**不会被复制的设置：**

-   背景图片
-   背景视频
-   背景 不透明度

## 2. 浏览器支持

**NewTab 项目仅官方支持最新版本的 Chrome 和 Firefox。**

开发和测试仅在 Chrome 和 Firefox 上进行。NewTab 在其他浏览器的性能和可靠性不在本项目范围内。

---

## 3. Cookie 和缓存

NewTab 不会收集、存储或发送您的任何数据。您的所有数据都保存在本地浏览器中。

这意味着如果你清除浏览器缓存或 cookies，NewTab 将会重置为其默认状态。

在关闭浏览器时清除缓存或 Cookie 的扩展程序或应用程序也会强制 NewTab 恢复到默认状态。


---

## 4. 数据备份与恢复

NewTab 是隐私优先扩展，不会将用户数据保存到远程数据库中。所有数据都保存在浏览器本地。

数据控制可在`菜单`  \> `Data` 中找到 Settings
### 1. 恢复数据



使用 Settings  `菜单`  \>  `数据`  \>  `还原`  \>  `从 文件导入`以恢复 NewTab 备份 JSON 文件。

NewTab 设置的`书签` 、 `主题`或`设置`可以从备份的 JSON 文件恢复。

-   选择`书签`后，会导入用户添加的书签，保留每个书签可能拥有的任何自定义颜色、重音和边框。
-   选择`主题`后，将导入颜色、强调、字体、背景以及所有保存的自定义主题。
-   选择`设置`后，将导入版面大小和位置、头部区域大小、书签区域大小及其他用户设置。

选择要恢复的备份 JSON 文件部分后，点击`导入`完成流程。



### 2. 备份数据



经常备份你的 NewTab 设置。这可以轻松地从 Settings `菜单`  \>  `数据`  \>  `备份`  \>  `导出数据` 。

备份的 JSON 文件会保存到你的设备上。该备份文件将以日期和时间戳命名，便于查找，例如 `NewTab backup - 2021.08.16 - 09 10 29.json` 。

你可以用这个备份文件在任何安装了 NewTab 的设备上恢复你的设置。

### 3. 数据重置



使用 Settings  `菜单`  \>  `数据`  \>  `清除`  \> 清除`所有数据`以重置 NewTab 到默认状态。

或者，把 NewTab 重置为默认状态，同时保留所有书签和群组。这可以通过 Settings `菜单`  \>  `数据`  \>  `清除`  \> 清除`除书签外的所有`内容。

## 5. 本地背景图像

不再支持上传图片作为背景。

这一变化是在 NewTab 7.0.0 中引入的

使用本地图像作为背景很有用，但也有许多缺点。

通常，NewTab 不会将任何用户数据存储在远程数据库中。所以上传并用作背景的图片文件必须存储在客户端的本地存储 （浏览器缓存中）。本地存储的大小限制 （因浏览器而异）大约是 5MB。

图片的大小很容易超过 5MB。当同一存储空间用来保存用户设置时，这就成了问题：布局、书签和主题设置。在这种情况下，浏览器可能会抛出错误，或者阻止 NewTab 正常运行。

从本地存储中移除几 MB 的图像数据也意味着 NewTab 读取保存数据的速度更快，从而缩短加载时间。

上传图片时的其他问题依然存在。有些图片就是加载失败。

基于上述原因，本地图像背景上传控制已被从功能集中移除。

要用图片作为背景，你应该用 URL 链接到图片 。只需将一个或多个 URL 粘贴到主题背景图像输入中即可更改背景图像。这种方法最可靠，且存储空间消耗最少。

## 6.项目目标

**该项目的目标总结如下：**

-   让浏览器的起始页面变暗（或亮色，或者介于两者之间）。
-   请勿访问用户信息。这意味着 NewTab 不会请求浏览器权限。
-   目标是在合理范围内让用户界面上的一切都可定制。
-   目标是让用户体验简单，同时支持上述功能。
-   保持 NewTab 开源。


---

## 7. 受保护的网址

Chrome 和 Firefox（以及其他浏览器）不允许网站或扩展打开某些 URL。这些特权地点包括：

-   以 chrome:// 开头的页面 `......``， 关于：...` 或 `edge： ...`
    -   例如 `chrome://history/`，`about：config` 或 `about：preferences`
-   链接到您设备上本地文件，URL 以 `file:/// 开头......`

出于安全原因，这些页面和网址无法通过应用程序或扩展访问。这就是为什么用特权浏览器 URL 添加到 NewTab 的书签无法打开。

该项目的目标之一是运行时无需额外 用户权限 。因此，目前没有计划访问浏览器中保存的用户书签。

---

## 8. 恢复设置和书签


如果出现错误或加载问题，使用 NewTab 时保存的用户设置和书签可以从浏览器 本地存储 中恢复。

1.  打开夜页
2.  打开开发者工具
3.  进入控制台 —— 查找浏览器的开发者控制台
4.  使用命令 `copy(localStorage.getItem("NewTab-backup"))` 或 `copy(localStorage.getItem("NewTab"))` 将数据复制到剪贴板
5.  将这些数据粘贴到一个空的文本文件中，并保存为 JSON 文件，例如 `my-NewTab-settings.json`。

---

## 9. 打开浏览器时重置

NewTab 会把所有设置、主题和书签保存到浏览器的本地存储中。这意味着如果浏览器缓存被手动清除或由其他应用程序清除，NewTab 可能会被重置为默认状态。

有些浏览器在关闭应用时可以清除缓存和 Cookie。这也可能导致 NewTab 重置为默认状态。

在试图找出导致 NewTab 重置为默认状态的原因时，可以尝试一个个禁用其他扩展或应用程序，看看问题是否依然存在。

NewTab 仅支持 Chrome 和 Firefox。这意味着在 Brave、Edge 或 Opera 等浏览器上，没有进行任何测试或开发以确保用户设置被正确保存。这超出了本项目的范围。

强烈建议用户定期备份 NewTab 设置 。

---

## 10. 尊重你的隐私

NewTab 完全在客户端浏览器中运行。用户数据不会被发送到任何地方，也没有远程数据库或中央用户管理。NewTab 不使用任何追踪码或 Cookies。

所有数据都保存在你使用的设备上。

## 11. 设置背景视频或图片

### 背景图片



背景图片可以在 Settings `菜单`  \>  `主题`  %`3E 背景`  \>  `图片`  \> `URL` 下设置 。

图片需要托管在网络某个地方。上传到 Imgur或 GitHub是托管文件的好选择。

### 背景视频

背景视频可以在 Settings`菜单`  \>  `主题`  \>  `背景`  \>  `视频`  \> `URL` 下设置 。

**YouTube 页面的链接不能用作背景视频。**

视频应该托管在网络某个地方。只需输入指向视频文件的 URL，视频背景仅支持 MP4 和 WebM 格式。

## 12. 将 NewTab 设置为你的 Firefox 主页


Firefox 不允许插件替代“主页”。不过有一个变通方法：

-   打开新的 Firefox 窗口
-   打开 Firefox `偏好设置` ，并打开一个新标签页（NewTab）
-   在 Firefox `首页`偏好设置中，更改`首页和新窗口`到  `自定义 URL......`
-   然后点击 `“使用当前页面` ”。

NewTab 现在将作为主页出现。
