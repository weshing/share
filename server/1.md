# 🌐 **Shing 产品规划（轻量版 AI 工具生态平台）**

## 🎯 1. 产品定位
**Shing 是一个开源的 AI 工具管理平台，提供二级域名、Web 管理界面与本地命令行工具，让用户可以创建、管理并分享自己的 AI 工具集合。**

Shing 不维护工具本身，而是提供：

- 工具仓库托管（用户自己的工具列表）
- Web 管理界面（可视化管理工具）
- 本地 CLI（安装、更新、卸载工具）
- 未来的桌面应用（图形化管理）

Shing 的核心理念：

> **用户自建工具生态，我们提供基础设施。**

---

# 🧩 2. 产品目标

### **2.1 让用户轻松管理自己的 AI 工具集合**
用户可以：

- 添加自己的工具  
- 添加团队内部工具  
- 添加 GitHub 上的开源工具  
- 自定义安装脚本  
- 自定义版本  
- 自定义分类  

### **2.2 让用户可以分享自己的工具生态**
每个用户拥有一个二级域名：

```
username.shing.sh
```

别人可以：

```
shing add username.shing.sh
shing install <tool>
```

### **2.3 让工具安装更简单、更统一**
本地 CLI 负责：

- 安装  
- 更新  
- 卸载  
- 同步工具列表  

---

# 🧱 3. 产品结构

## **3.1 本地命令行工具（核心）**
用户通过 CLI 安装工具：

```
shing install llama.cpp
shing install whisper
shing install my-custom-tool
```

CLI 功能：

- 读取用户仓库的工具元数据  
- 下载工具（GitHub Releases / URL）  
- 执行安装脚本  
- 管理版本  
- 管理卸载  

CLI 是整个系统的执行引擎。

---

## **3.2 用户专属二级域名**
每个用户拥有一个工具仓库：

```
username.shing.sh
```

仓库内容包括：

- 工具列表  
- 工具元数据  
- 工具版本  
- 安装脚本  
- 分类与标签  

我们提供托管与 CDN。

---

## **3.3 Web 管理界面**
用户可以在 Web 上：

- 添加工具  
- 编辑工具信息  
- 配置安装脚本  
- 设置 GitHub Releases 源  
- 管理版本  
- 管理分类  
- 管理标签  

Web 是用户的“工具后台”。

---

## **3.4 桌面应用（未来）**
桌面应用提供：

- 图形化工具管理  
- 一键安装  
- 一键更新  
- 一键卸载  
- 同步用户仓库  

底层仍然调用 `shing` CLI。

# 🚀 5. 平台价值（Shing 提供什么）

## **5.1 工具仓库模板（manifest 规范）**
例如：

```yaml
tool:
  name: llama.cpp
  version: 0.3.1
  os: [macos, linux]
  arch: [x86_64, arm64]
  source: https://github.com/ggerganov/llama.cpp/releases/download/v0.3.1/llama.tar.gz
  install:
    - tar -xzf llama.tar.gz
    - mv llama /usr/local/bin
```

---

## **5.2 Web 管理界面**
用户不需要写 YAML，可以用 UI 管理。

---

## **5.3 二级域名托管**
我们提供：

```
username.shing.sh
```

用户的工具仓库自动部署到 CDN。

---

## **5.4 本地命令行工具（shing）**
负责：

- 下载  
- 安装  
- 更新  
- 卸载  
- 同步仓库

# 🧭 6. 产品边界（保持轻量）

Shing **不做**：

- ❌ 不维护官方工具列表  
- ❌ 不做模型下载  
- ❌ 不做 GPU 驱动管理  
- ❌ 不做环境管理（conda/nvm）  
- ❌ 不做推理引擎  
- ❌ 不做插件系统  
- ❌ 不做生态平台  

Shing **只做**：

> **用户自建工具生态的平台 + 本地工具安装器**
